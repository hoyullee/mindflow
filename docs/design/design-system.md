# Geurio 디자인 시스템 (v1)

> **목적** — `apps/web` 곳곳에 인라인 스타일로 흩어진 색·타이포·간격·둥근모서리·카드
> 규격을 **하나의 기준**으로 모읍니다. 앞으로 "카드가 너무 커/작아", "타이틀이 안 띄어"
> 같은 디자인 요청이 오면 **감이 아니라 이 문서의 토큰**으로 답을 결정합니다.
>
> 이 문서는 **사람과 에이전트(특히 `ux-ui-designer`, `react-porter`)가 공유하는 단일
> 디자인 컨텍스트**입니다. 새 UI를 만들거나 고칠 때 먼저 읽습니다.
>
> **상태**: 서술형 카탈로그(v1). 값은 현재 코드에서 **역으로 정리**한 것이며, 아직 CSS
> 변수/토큰 파일로 강제하지는 않습니다(대규모 리팩터 위험 회피). 새 코드와 수정은 이
> 문서의 토큰명을 **주석·근거로 인용**하고, 값이 어긋나면 이 문서를 갱신합니다.

## 1. 브랜드 정체성

Geurio의 톤은 **따뜻한 코럴(coral) + 웜 그레이(warm gray)** 입니다. 차갑지 않고
친근하며, 종이·손그림 마인드맵의 감성을 지향합니다.

- **브랜드 마크**: 코럴 둥근사각 + 흰 "G" (arc + crossbar). 아이콘 생성기
  `apps/web/scripts/generate-icons.mjs`, `apps/mobile/scripts/generate-native-assets.mjs`.
- **기본 테마**: 에디터의 `coral` 테마(`src/features/editor/theme.ts`)가 앱 전체 크롬의
  기준 팔레트입니다. 아래 색 토큰은 대부분 coral 테마에서 파생됩니다.

## 8. 카드 규격 (그동안 가장 많이 흔들린 곳 — 여기서 고정)

> **배경**: PR #106~#123 사이에서 카드 폭이 104→128→132→128px를 오가고, 좌우로 늘어나는
> 문제가 반복됐습니다. **근본 원인은 CSS Grid `1fr`(= `minmax(auto,1fr)`)** 입니다 —
> `auto` 바닥값이 트랙을 콘텐츠(긴 제목)만큼 늘려 카드가 불균등·과폭이 됩니다.
> **결론**: 카드는 **절대 `1fr`로 늘리지 않는다.**

### 8.1 규칙
1. **최근 항목 카드**: **고정 폭** `RECENT_CARD_W = 128px`. 뷰포트에 따라 바뀌는 것은
   **개수(cols)** 뿐, 개별 카드 크기는 절대 안 바뀜.
   (`src/features/home/components/RecentStrip.tsx` — ResizeObserver로 폭 측정 →
   `cols = clamp(floor((w+gap)/step), 2, 8)`, `gridTemplateColumns: repeat(shown, 128px)`.)
2. **맵/폴더 카드(데스크톱)**: `grid-template-columns: repeat(auto-fill, minmax(300px, 1fr))`.
   여기서 `1fr`은 **행을 꽉 채우는 균등 분배용**으로 허용(최소 300px 보장 + 균등). 카드가
   하나든 여럿이든 같은 크기가 되도록 `auto-fill`을 씁니다.
3. **모바일(≤767px)**: 1열 `minmax(0, 1fr)`. **`0` 하한**이라 긴 제목이 트랙을 밀어
   가로 오버플로를 만들지 못함. (bare `1fr`=`minmax(auto,1fr)` 금지.)
4. **썸네일 비율**: 픽셀 높이 대신 `aspect-ratio`로 고정 → 모바일 풀폭에서도 PC와 같은 비율.
   - 맵 카드 썸네일 `304 / 150`
   - 최근 카드 썸네일 `148 / 72`
5. **긴 제목**: 카드 폭을 늘리지 않고 **말줄임(ellipsis)**. 제목 길이는 레이아웃에
   영향 주지 않음.
6. **레이아웃 시프트 방지**: 스크롤바 등장 시 밀림 방지로 `scrollbar-gutter: stable`.

### 8.2 방어 코드 (회귀 가드)
- `RecentStrip` 컨테이너 `overflow: hidden` — 하위 최소폭 넘침 백스톱.
- `home.css`의 `.mf-recent-grid` 사전측정 폴백도 **고정폭** `repeat(auto-fill, 128px)`
  (절대 `1fr` 아님).
- **테스트**(`Home.test.tsx`): `.mf-recent-grid`의 `gridTemplateColumns`에 `px`가 있고
  `fr`이 **없음**을 단언 → `1fr` 회귀를 CI에서 잡음.

## 9. 반응형

- **브레이크포인트**: `768px`. `@media (max-width: 767px)`가 모바일.
- 모바일 변형(참고): 홈 사이드바→드로어, 에디터 속성패널→바텀시트, 최소 44px 터치타깃.
- `@media (hover: none)`에서 hover로만 뜨던 카드 컨트롤(☆·☰)은 **항상 표시**.
- `@media (prefers-reduced-motion: reduce)`에서 스켈레톤 애니메이션 중지.

## 10. 아이콘

- **인라인 SVG 우선**. 이모지(🗺️/⭐/🗑️ 등)는 플랫폼마다 렌더가 달라 UI 일관성을 해치므로
  **본문·기능 아이콘에는 쓰지 않습니다**(PR #109~#113에서 SVG로 교체 완료).
- 도형 스타일 미리보기는 `panelPrimitives.tsx`의 `ShapeGlyph`, LNB 아이콘은
  `Sidebar.tsx`의 `StarGlyph`/`TrashGlyph`.
- SVG는 `currentColor`로 색을 상속받게 그려 시맨틱 토큰과 어울리게 합니다.

## 11. 사용 규칙 요약 (에이전트·사람 공통)

1. 새 UI 색은 **§2 토큰에서 고른다.** 새 hex를 즉흥으로 만들지 않는다. 부득이하면 이
   문서에 토큰으로 먼저 추가한다.
2. 텍스트 크기·두께는 **§3 스케일 안에서** 고른다.
3. 카드는 **§8 규칙**을 따른다. 특히 **최근 카드는 고정폭, `1fr` 금지.**
4. 모바일 터치 타깃 **44px 이하 금지**.
5. 기능 아이콘은 **SVG**(이모지 금지).
6. 값이 이 문서와 어긋나게 바뀌면 **코드와 이 문서를 함께 갱신**한다(문서가 뒤처지면
   신뢰를 잃는다).
